PHP 4.2.2 以上 必須。
mbstring 必須。－これが無いと絶対に動作しません。

Redhat Linux 9.0
Apache version 1.3.31
PHP 4.3.3
Xoops 2.0.6

上記の環境で動作確認。インストール－アンインストールまで。

複雑な機能は一切ありません。特定の商品をピックアップしたり、特定の商品
をブロックに表示する機能もありません。勝手に表示します。
ブロックに表示される商品は、各ベストセラー商品上位２品を集め、ランダム
に表示します。最初に開くページも勝手に決めます。

管理ページの一般設定よりアソシエイトID、発行されたトークン、キャッシュ
場所キャッシュ更新時間（時間）、キャッシュを消す時間－作成からの経過時
間、WSDLの位置を指定して下さい。
特に、アソシエイト ID　発行されたトークン　の変更を忘れないで下さい。
WSDL はURLを指定するより保存されたものを呼び出す方が効率的です。
たまに更新されるので時々入れ替えて下さい。※めったに変更されません。

index.php
define( "_AM_DEFAULT_KEYWORD", 'xoops' );
これは、特定のキーワードを入れて下さい。
最初に呼び出されたときに、
☆和書 ★音楽 ☆DVD ★ホーム&キッチン ☆コンピューター　のベストセラー
このキーワード検索の結果をランダムに表示します。

AMAZON キャッシュ更新時間（時間）は上記の内容を保持し、指定時間が経過し
ていれば再度取得すると言う仕組みです。（２４時間くらいを推奨します）
また、＜＜　取り上げてみる　＞＞で取り込まれたデータもこの時間キャッシュ
されます。
キーワードで検索された内容も一応キャッシュできますが、再使用されることは
稀です。（OFF にしています）
上記ランダムを利用して「作成からの経過時間」を経過したキャッシュファイル
を削除します。消す作業はコストがかかるのでランダムに仕事させています。
※最大で3Mほどたまったことがあります。（自分でTESTしてためてみた）

AMAZON キャッシュ更新時間　＜　キャッシュを消す時間　として下さい。

※キャッシュを消す時間が小さいと絶えずアクセスが発生し、サイトのパフォー
マンスが落ちます。
また、AMAZONWeb サービスもアクセス制限があるので結果を取得出来ないことに
なってしまいます。

Templateも使用しないのでほとんど自由度がありません。またこれ以上の機能追
加の予定もありません。
AMAZONWeb サービスがバージョンアップを予定しているのでその時には対応する
かもしれません。
